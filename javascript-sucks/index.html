Every now and then I find out something really awful about javascript. I also often forget them though. This is a place where I write down things I think really suck about js. ## Issues __Array.sort__ does comparisons by converting each element into a string. So if you sort `[5, 10]`, you get `[10, 5]`. WTF. __Array equality__ is by object identity. So `[1, 2] === [1, 2]` is false. WTF. This could potentially be mitigated with an 'equals' method like in Java, but as of ES6, it still seems like developers still have to resort to manual means of comparing equality: http://stackoverflow.com/questions/7837456/how-to-compare-arrays-in-javascript ## Issues mitigated in ES6 __Object keys are stringified__ The major issue with this was that there was no 'Set' or 'Map' objects where you can use arbitrary keys/hash. This is mitigated in ES6 with actual 'Set' and 'Map' methods, but they are still plauged by the old equality semantics.
